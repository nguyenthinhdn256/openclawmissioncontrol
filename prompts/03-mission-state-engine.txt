Bạn đang xử lý PHẦN 3/8: MISSION STATE ENGINE cho project OPENCLAW MISSION CONTROL.

MỤC TIÊU PHẦN NÀY
Tạo engine quản lý lifecycle mission theo kiểu deterministic.

ĐƯỜNG DẪN FILE PHẢI DÙNG
- src/lib/state/mission-state-machine.ts
- src/lib/state/mission-selectors.ts
- src/lib/state/mission-actions.ts
- src/lib/state/index.ts
- src/docs/mission-state-engine-notes.md

PHỤ THUỘC ĐẦU VÀO
- Mission và MissionStatus từ `src/lib/contracts`
- Không tự tạo schema mới nếu phần contracts đã định nghĩa

YÊU CẦU CHI TIẾT
- Tạo bảng transition hợp lệ
- Hàm `canTransitionMission`
- Hàm `transitionMission`
- Hàm `computeMissionProgress`
- Selector đếm mission theo status
- Selector lấy mission overdue / blocked / ready nếu cần mock-based
- Action creators dạng thuần object, không cần Redux

RÀNG BUỘC
- Invalid transition phải trả structured error rõ ràng
- Không dùng external state library
- Logic phải độc lập, testable
- Phải dễ cho UI đọc summary

ĐẦU RA BẮT BUỘC
- Code đầy đủ
- Mô tả engine này sẽ được phần dashboard và qa dùng như thế nào
